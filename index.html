HTML

  • HTML is Hypertext Markup Language and is used to create the structure of a webpage
  • An element is a type of content on a webpage
  • Text, Image, Header, and List elements are commonly used to add content to pages
  • Meta-related, Content division, and Semantic elements are used to divide and describe a page
  • When developing a webpage, first, we use HTML elements as the building blocks of our sturdy frame
  • HTML elements are case-sensitive, while declarations are not case-sensitive
  • Metadata is data about a webpage that the browser uses to effectively server the webpage to users
  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • The title element defines the title for a webpage. It is required and is important for Search Engine Optimization(SEO)
  • The style element defines the CSS styles associated with a website
  • The base element defines the base URL for a webpage
  • The link element connects an external resource to the HTML document
  • The meta element defines metadata such as the character set. description, keywords, author, and viewport
  • The script element embeds client-side scripts in the HTML document
  • Elements that are nested inside other elements are called child elements
  • The parent element is the element that holds all the children
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage
  • Non-semantic elements such as 'div' and 'span' are used to create division
  • When working with visible HTML elements, the content is contained between an opening tag and a closing
  • Difference between element and tag. An element refers to the entire element from opening tag to closing tag; whereas a tag refers only to what appears inside the angle brackets
  • The 'h1', 'h2', 'h3', 'h4', 'h5', and 'h6' elements represent the level of heading a given text block represents
  • The 'p' element represents a paragraph or block of text
  • The 'u1', 'o1', and 'li' elements represent unordered lists, ordered lists, and list items
  • The 'img' element contains information about images that are displayed on the webpage
  • The 'img' element can contain various attributes; the 'src' attribute is the most important because it defines the location of the image
  • The 'br' element creates a line of empty space, or a line break, between two blocks of content
  • The 'a' element (which stands for anchor) creates links to the same or other webpages
  • The 'href' atttribute points to the URL for the link
  • The 'src' attribute defines the location, or source of the image file
  • The 'alt' attribute contains a text string that describes the appearance and functionality of the image
  • The footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation
  • Section elements are semantic elements that define a section in a document that contains thematic content
  • The 'id' attribute is used as a way to identify elements and select them (class attribute works similarly)
  • Developers test code early and often to catch bugs; sometimes this can be as simple as looking for changes in the browser to see if a webpage is rendering as expected

    Git

  • Git is version control for developers
  • Every project on GitHub is called a repository (repo), a folder for your project that contains all of the files and their revision history
  • commit changes: saving a new version of a file you are working on
  • pull request (PR): asking for changes on a branch to merge into another branch
  • merge: when a PR is approved, changes are pulled in or merged into main branch
  • Issue: a way to report new tasks or content fixes
  • When naming a repository, remember to: be unique and descriptive for each repo name, use all lowercase letters, use dashes instead of spaces, and be consistent
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Branch: workspace where you can make changes that don't affect the live site
  • Markdown(.md): the way content files are formatted, in plain text, which easily converts to HTML
  • git add -A: the add command adds modifications in the current working branch to the staging area and the -A flag indicates that we want to add all changes
  • git commit -m "commit message describing what the commit contains": commit created that captures a snapshot of the currently staged changes
  • git pull origin main: used to make sure local branch is in sync with base branch in GitHub
  • It is important to always to a git pull to make sure that you have the latest version of the projext on your local computer before you start to work
  • git push origin branch-name: pushes changes from local branch to remote branch
  • code .:opens current directory in VS Code
